# XML v11

The XML builder follows the Cape Verde e-Fatura Technical Manual v11.0 for the DFE envelope shape.

## Envelope

Every generated DFE XML uses a compact UTF-8 XML 1.0 document with the root element below:

```xml
<Dfe xmlns="urn:cv:efatura:xsd:v1.0" Version="1.0" Id="{IUD}" DocumentTypeCode="{code}">
```

The package does not add namespace prefixes, comments, or formatting whitespace.

## Document Element

Manual v11.0 requires the first child of `Dfe` to be the document-specific element. The current mapping is:

| Code | Acronym | Element |
|------|---------|---------|
| 1 | FTE | `Invoice` |
| 2 | FRE | `InvoiceReceipt` |
| 3 | TVE | `SalesReceipt` |
| 4 | RCE | `Receipt` |
| 5 | NCE | `CreditNote` |
| 6 | NDE | `DebitNote` |
| 7 | DTE | `Transport` |
| 8 | DVE | `ReturnNote` |
| 9 | NLE | `RegistrationNote` |

`buildDfeXml()` and `efatura.buildDfeXml()` use this mapping through `dfeDocumentElementName()`.

## Common Blocks

The document element contains the package-supported common blocks:

- `Identification`: LED, serie, document number, issue date, issue time, issue mode, due date, tax point date, issue reason, and isolated act flag.
- `Software`: registered software code, name, and version.
- `Emitter`, `Receiver`, and optional `Payer`.
- `Lines`, including tax details.
- `Totals`.
- `Reference`, for credit, debit, and return note references.
- `Contingency`, when `IssueMode` is `OFFLINE` or `OFF`.
- `extraFields`, serialized as additional XML blocks for legal fields not yet typed as first-class value objects.

## Official Validation

The manual states that systems must validate generated XML against the official XSDs published by the tax administration before submission. Those XSD files are external artifacts. This package exposes the `XsdValidator` contract and uses `MissingOfficialXsdValidator` by default, which fails explicitly until an official validator is injected.

```ts
const efatura = createEfatura(config, {
  xsdValidator: officialValidator,
});

await efatura.validateDfeXml(xml, DocumentType.ElectronicInvoice);
```
